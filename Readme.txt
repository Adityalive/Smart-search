**Phase 1 — Foundation.** This is your skeleton. Set up a React frontend with Vite. Build a Node/Express backend with routes like `POST /api/items` (save a link) and `GET /api/items` (fetch all). Connect MongoDB using Mongoose. Add user login/signup with JWT. By the end, a user can register, log in, and save a URL that appears on their dashboard.

**Phase 2 — Content extraction.** When a URL is saved, your backend uses Cheerio to scrape the page — extracting the title, description, and main text. Users can upload PDFs too (Multer handles file uploads). They can also create collections (like folders) and manually add tags. By the end, saved items have real content, not just a bare URL.

**Phase 3 — AI tagging + semantic search.** This is the biggest leap. When an item is saved, instead of processing it immediately (which would slow down the API response), you push a job to a BullMQ queue. A background worker picks it up, sends the content to the OpenAI API, gets back suggested tags, and generates a vector embedding. The embedding is stored in Qdrant. Now your search endpoint can do semantic search — searching for "AI learning" finds "neural networks tutorial" even without exact keyword overlap.

**Phase 4 — Knowledge graph + related items.** Using the embeddings already stored in Qdrant, compute cosine similarity between items. Items that score above a threshold are "related." Build a `GET /api/items/:id/related` endpoint. On the frontend, render a D3.js force-directed graph where nodes are saved items and edges are similarity connections. Also show a "Related items" panel when viewing any item.

**Phase 5 — Chrome extension + advanced features.** Build a Manifest V3 Chrome extension with a single "Save to App" button. It reads the current tab's URL and title, then calls your backend API. Add the highlight system (store highlighted text ranges in MongoDB). Add memory resurfacing using `node-cron` — a scheduled job that checks for items saved 30/60/90 days ago and surfaces them.
